# ADR-260057: Centralized GraphQL Error-Code Mapping, Authenticated Reads, and Rate-Limit Handling

- **Date**: 2026-06-23
- **Status**: Accepted

## Context

8.1.0 changes both **what** is authorized and **how** errors are transported. Almost every read —
including taxonomy reads that used to be public — now requires a Bearer session token; the public set
is limited to `signin`, `signinGoogle`, `schemaInfo`, `discoverOperations`, `beginSignup`,
`completeSignup`, `requestPasswordReset`, `confirmPasswordReset`, `logout`. Error codes now arrive in
the GraphQL `errors[].message` (no `extensions.code`, no error body on data fields). Sign-in, code
entry, and code send are rate-limited (`AUTH-RATE-LIMITED`). The client currently has no single place
that interprets error codes, and its expiry detector reads `extensions.code`.

## Decision

1. **Central error-code mapper (D-21):** add one `api-contract` utility that parses
   `errors[].message` → a known error-code enum → a typed, user-facing outcome. Every feature consumes
   it; no feature parses raw messages. It is the backbone for sign-up (ADR-260055), session expiry
   (ADR-260056), Google linking and rate limiting (this ADR), and account management (ADR-260058).
   Coverage is required for **every** code in the epic error table.
2. **Authenticated reads (D-20):** the Apollo auth link already attaches the Bearer token to all
   operations; verify that **no data/taxonomy read runs before a token exists**. `schemaInfo` stays
   public (startup handshake); all workspace and taxonomy reads stay behind the auth guard.
3. **Rate-limit handling (D-17):** on `AUTH-RATE-LIMITED`, show a generic "too many attempts, try again
   shortly" message with submit back-off; never reveal whether an email exists.

## Alternatives Considered

- **Per-feature error parsing:** rejected — duplicative, drifts, and is the current source of the
  silent-expiry bug.
- **Trusting `extensions.code`:** rejected — 8.1.0 does not populate it for these codes.

## Consequences

- **Positive**: one tested translation layer; consistent UX; removes the silent-expiry class of bug.
- **Negative**: a new shared contract all auth/authz features depend on.
- **Risks**: unmapped/changed codes fall through; mitigated by a default "unexpected error" branch +
  exhaustive table coverage tests.

## Verification

- Unit: mapper covers every code in the epic table incl. unknown-code fallback; startup ordering test
  asserts no gated read before token. E2E: gated call without token routes to sign-in;
  rate-limit message shown. 100% coverage of the mapper and read-gating paths.

## Rollback

- Revert features to local handling (re-introduces drift); not recommended.
