# ADR-260023: Blank-First Workspace Entry Supersedes Bootstrap Search on Load

## Workflow

- This ADR records the accepted update to workspace entry behavior after observing ambiguity and trust issues caused by implicit graph preload.
- This ADR supersedes only decision C1 from ADR-260014; decisions C2-C7 remain active.

Status: Accepted
Date: 2026-05-08
Owners: Repository owner, product collaborator

## Context

ADR-260014 originally required a bootstrap search on workspace load to avoid an empty canvas. Observed usage indicates this behavior makes it unclear whether visible atoms come from explicit user intent or an implicit default query.

The project has already accepted requirement updates for blank-first workspace entry (`REQ-FR-260022`) and deprecated the bootstrap-search requirement (`REQ-FR-260007`).

## Decision

- Workspace entry model update accepted:
  - After sign-in, the workspace opens in a blank graph state with no atoms/bonds rendered.
  - Graph content appears only after an explicit user search action.
- Supersession scope:
  - Supersedes ADR-260014 decision C1 ("graph-first with search bootstrap on load").
  - Does not supersede ADR-260014 decisions C2-C7.

## Alternatives Considered

- Option A: Keep bootstrap search on load and tune default query.
- Option B: Remove bootstrap search and require explicit search before graph population.

## Consequences

- Positive:
  - Aligns first visible graph state with explicit user intent.
  - Reduces ambiguity about where initial graph contents came from.
- Negative:
  - Blank entry can feel empty without clear search affordances.
- Risks:
  - If search affordances are weak, first-use confidence may drop until search-model refinements land.

## Verification

- Verify sign-in lands in a blank workspace with no preloaded atoms.
- Verify graph population occurs only after explicit search submission.
- Verify route/auth stability is unchanged from prior entry flow.

## Rollback

- Supersede this ADR if evidence shows blank-first entry harms first-use success more than bootstrap ambiguity.
- Restore bootstrap behavior only through a new superseding ADR and linked requirement updates.
