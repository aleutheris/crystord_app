# ADR-260022: Branding Governance as a First-Class Constraint

## Workflow

- This ADR records the accepted branding-governance decision for ongoing UI/design work.
- The decision is accepted to ensure branding consistency remains explicit, traceable, and enforceable during direct-to-main delivery.

Status: Accepted
Date: 2026-04-17
Owners: Repository owner

## Context

The project already has a detailed branding brief in `docs/governance/project/branding.md`, but branding governance was tracked as BI-260019 without a linked ADR or requirement.

Without a formal architecture-level decision and requirement link, branding can become optional during implementation and review, especially in a solo, direct-to-main workflow.

## Decision

- Branding governance is formalized as an architectural and delivery constraint.
- `docs/governance/project/branding.md` remains the operational source of truth for UI semantics, token usage, and design constraints.
- Any UI/design-facing change must show alignment with branding semantics during implementation and main-branch checklist review.
- Branding governance traceability is represented by:
  - ADR-260022 (decision authority)
  - REQ-CR-260011 (constraint requirement)
  - BI-260019 (delivery slice)

## Alternatives Considered

- Option A: Keep branding only as an informal design brief with no ADR/requirement traceability.
- Option B: Move all branding detail into ADRs and remove a dedicated branding brief.
- Option C: Keep a dedicated branding brief and add ADR/requirement traceability.

## Consequences

- Positive:
  - Branding policy becomes auditable and enforceable like other core constraints.
  - Review quality improves because semantic color and interaction intent are explicitly checked.
  - Traceability consistency is restored in backlog and requirements indexes.
- Negative:
  - Adds one more governance artifact to maintain.
- Risks:
  - Branding drift remains possible if checklist discipline is not followed.

## Verification

- Governance files contain consistent references among BI-260019, ADR-260022, and REQ-CR-260011.
- Main branch checklist includes branding alignment as a mandatory check.
- Project instructions and loading matrix keep branding policy discoverable.

## Rollback

- Supersede this ADR only if branding governance is replaced by a stronger and equally traceable mechanism.
- If superseded, update REQ-CR-260011 and BI-260019 links accordingly.
