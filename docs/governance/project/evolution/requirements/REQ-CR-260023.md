# Requirement REQ-CR-260023: Client-Side Username and Password Validation Parity

- **Status**: Proposed
- **Description**: The client MUST enforce, via a single shared validation schema reused by sign-up
  (`completeSignup`), `setPassword`, and password reset (`confirmPasswordReset`):
  - **Username**: 3–30 chars; letters/digits/`.`/`_`/`-` only; must start with a letter; no
    leading/trailing or consecutive separators; not reserved (`admin`, `support`, `api`, `me`, `root`,
    `system`).
  - **Password**: minimum 12 chars, maximum 72 bytes, not a common password.
  The server remains authoritative — `USER-INVALID-USERNAME`, `PASSWORD-TOO-SHORT/-LONG/-COMMON` MUST
  still be handled if returned.
- **Rationale**: Mirrors server rules to avoid round-trips and give immediate inline feedback while
  keeping the server as source of truth.
- **Related ADRs**: [ADR-260055](ADR-260055.md), [ADR-260058](ADR-260058.md)
