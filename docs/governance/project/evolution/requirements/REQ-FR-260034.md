# REQ-FR-260034: Dual-View Validation Coverage Must Include Unit, Integration, and E2E Paths

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-graph, workspace-search, ui-shell
**Affects**: test coverage policy for dual-view behavior
**Related ADRs**: ADR-260032
**Related Requirements**: REQ-FR-260030, REQ-FR-260031, REQ-FR-260032, REQ-FR-260033

## Summary

Dual-view delivery shall include defined unit, integration, and E2E coverage for switching, shared-state consistency, large-result gating, and keyboard interaction paths.

## Specification

1. Unit tests cover tab state transitions, selection synchronization, threshold gating, and keyboard handlers.
2. Integration tests cover shared dataset parity, mutation reflection, and degrade-threshold transitions.
3. E2E tests cover default Network view entry flow, cross-view selection carryover, >400 continue gate, and keyboard-only path.

## Rationale

Multi-layer tests are required to prevent regression across state, rendering, and route-level behavior.

## Out of Scope

- Exhaustive cross-browser matrix beyond current MVP quality policy.
