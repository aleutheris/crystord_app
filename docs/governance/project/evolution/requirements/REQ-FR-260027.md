# REQ-FR-260027: No Graph Re-Scoping While Search Input Is Being Edited

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-search
**Affects**: SearchBar.tsx, use-search.ts, workspace graph search integration
**Related ADRs**: ADR-260016, ADR-260023
**Related Requirements**: REQ-FR-260024, REQ-FR-260026, REQ-CR-260013

## Summary

Editing the search input must not change the graph or search-result presentation until the user explicitly submits the search.

## Specification

1. Typing characters into the search input must not re-scope the graph.
2. Adding or removing partial text before submission must not trigger graph updates.
3. Graph changes related to search are permitted only after explicit submission using the accepted search action.
4. Existing internal structure for future client-side re-scope behavior may remain in the codebase, but it must not be enabled in the current product behavior.

## Rationale

- Prevents misleading motion and false confidence while the user is still composing intent.
- Keeps visible graph state aligned with explicit backend-backed search execution.
- Preserves the option to reintroduce client-side refinement later without committing to it now.

## Out of Scope

- Future opt-in local refinement after an initial backend search.
- Animated incremental graph previews while typing.

## Implementation Notes

- Disable wiring that applies search-state changes on keystroke.
- Preserve reusable search-state abstractions where they help future iteration, but keep them inactive.
- See BI-260025 for implementation scope.
