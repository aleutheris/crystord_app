# REQ-FR-260005: Backend Contract for Essential Atom and Bond Operations

- Class: FR
- Status: Active
- Date: 2026-04-13
- Source decision: B1, B2, B4 (Backend contract and operations)

## Requirement

The frontend shall implement retrieve, change (create/update), and destroy mutations for Atoms and Bonds, binding to the GraphQL schema documented in the backend user guide.

## Rationale

These core operations map directly to MVP data-management workflows and establish the contract boundary between frontend and backend.

## Scope Notes

- In scope for MVP: retrieve atoms by UUID and labels, create and update atoms, create bonds through drag-to-connect, delete atoms, login via signin with fixed demo credentials in development/testing.
- Out of scope for this requirement: label creation, operations evaluation, constants management, and field-level authorization.

## Verification

- Contract tests confirm retrieve returns expected Atom structure with labels, bonds, and nuclearies.
- Contract tests confirm change mutation succeeds for atom create/update/bond create.
- Contract tests confirm destroy mutation succeeds and removes atom from subsequent queries.
- Integration tests confirm demo sign-in works in development/testing runtime.
