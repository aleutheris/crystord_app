# REQ-OR-260009: Bond Mutation Client Contract Must Match the Schema Selector Type

- Class: OR
- Status: Accepted
- Date: 2026-05-10
- Source decision: Bond creation contract alignment after POST 400 observation

## Requirement

For atom bond creation and deletion flows, the client GraphQL documents shall use the schema-defined `Selector` type and the approved mutation shape exactly as published in the backend contract files.

## Rationale

The bond workflow failed because the client-side mutation document used a non-existent selector type name (`SelectorInput`) instead of the schema contract type (`Selector`). Keeping the client document aligned with the schema prevents avoidable 400-level contract failures and preserves the bond dialog UX.

## Scope Notes

- In scope for this requirement: bond create/update/delete mutation documents and their variable type names.
- In scope for this requirement: keeping the bond naming dialog unchanged while fixing the GraphQL contract.
- Out of scope for this requirement: changing bond naming UX, bond semantics, or backend business rules.

## Verification

- Contract check confirms client mutation documents use `Selector` rather than `SelectorInput`.
- Runtime check confirms bond operations execute against the approved schema without request-shape rejection.
- Regression check confirms the bond dialog remains a simple name entry step.
