# REQ-FR-260024: Search Execution on Enter Key

**Status**: Accepted
**Type**: Functional Requirement
**Module**: workspace-search
**Affects**: SearchBar.tsx, use-search.ts
**Related ADRs**: ADR-260023
**Related Requirements**: REQ-FR-260025

## Summary

Search should execute only when the user explicitly presses the Enter/Return key, not on every keystroke or intermediate state change.

## Specification

The search control must NOT trigger live filtering as the user types. Instead:

1. User composes search input (text and/or label chips)
2. User presses Enter/Return key or clicks the submit button
3. Search query executes against the backend (fresh query, not client-side filtering)
4. Results render in the workspace

## Rationale

- **User intent clarity**: Enter key provides explicit commit signal; keystroke-based execution can feel unpredictable or overwhelming
- **Backend freshness**: Defers search execution until user is ready, avoiding redundant queries
- **Accessibility**: Matches standard form submission pattern (Enter to submit)

## Out of Scope (Deferred)

- Autocomplete / incremental suggestions (see REQ-CR-260013)
- Keyboard shortcut customization

## Implementation Notes

- Bind the form submit handler to both Enter and the visible submit button
- Suppress keystroke-based filtering behavior
- See BI-260023 for implementation task
