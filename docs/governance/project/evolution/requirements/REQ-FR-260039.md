# REQ-FR-260039: Network View Connection Drag Preview Must Render as a Straight Line

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view connection drag-preview rendering during bond creation gesture
**Related ADRs**: ADR-260037
**Related Requirements**: REQ-FR-260038, REQ-FR-260035, REQ-FR-260036

## Summary

In Network view, while the user is actively dragging to create a relationship from the outer initiation ring, the temporary preview edge shall render as a straight line.

## Specification

1. During active connection drag in Network view, the preview edge uses a straight-line segment from source to current pointer/target position.
2. Preview styling communicates direction affordance consistently with established direction semantics, without changing bond direction rules.
3. Established edges remain governed by existing nearest-boundary anchoring and arrowhead requirements.
4. Flow view behavior remains unchanged.

## Rationale

A straight preview line makes edge-creation intent easier to predict and reduces visual ambiguity during gesture completion.

## Out of Scope

- Changes to stored bond data contracts.
- Changes to established-edge anchoring algorithm.
- Touch-specific gesture redesign beyond existing desktop-first scope.
