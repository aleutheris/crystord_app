# REQ-CR-260016: Dual-View Tabs Must Follow ARIA Tab Pattern with Visible Focus

- Class: CR
- Status: Accepted
- Date: 2026-05-12
- Source decision: dual-view tab switching requires explicit semantics and focus clarity

## Requirement

The dual-view switcher shall implement standard ARIA tab semantics and provide visible focus indicators for tab controls and graph focus targets.

## Rationale

Clear semantics and focus visibility are required for assistive technologies and keyboard users.

## Scope Notes

- In scope for this requirement: `tablist`/`tab`/`tabpanel` semantics, selected-state linkage, visible keyboard focus.
- Out of scope for this requirement: custom non-standard tab semantics.

## Verification

- Accessibility check confirms tab semantics and selected-state attributes are correct.
- Keyboard check confirms focus indicator is consistently visible across tabs and graph focus targets.
