# REQ-FR-260010: Direct-Manipulation Primitives and Canvas-First Creation

- Class: FR
- Status: Active
- Date: 2026-04-17
- Source decision: D3, D4, D7 (Interaction primitives, creation flow, editing semantics)

## Requirement

The frontend shall support direct-manipulation graph operations for selection, atom movement, drag-to-connect bond creation, pan/zoom navigation, and search focus/center, and shall implement a canvas-first creation flow with side-panel completion for metadata-heavy edits.

## Rationale

Core graph workflows require direct spatial interaction in canvas while metadata complexity is better handled in side panels.

## Scope Notes

- In scope for MVP: required direct primitives, canvas quick-add, side-panel detail completion, immediate bond naming follow-up.
- Out of scope for this requirement: lasso/multi-select power workflows, advanced shortcut suites, full modal wizards.

## Verification

- End-to-end checks confirm each required primitive is available and usable.
- Workflow checks confirm atom creation starts in canvas and continues in side panel.
- Workflow checks confirm bond creation is drag-connect with immediate naming follow-up.
