# REQ-FR-260042: Network View Selected-State Must Not Force Outer Initiation Ring Visibility

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view node visual-state semantics for selection, hover, and connection initiation affordance
**Related ADRs**: ADR-260038
**Related Requirements**: REQ-FR-260038, REQ-CR-260017

## Summary

In Network view, node selection state shall be conveyed by inner selection visual semantics and must not, by itself, force outer initiation ring visibility.

## Specification

1. Outer initiation ring is visible on hover.
2. Outer initiation ring is visible during active connection initiation/drag affordance state.
3. Outer initiation ring is not shown solely because a node is selected.
4. Inner selection visual remains the selected-state indicator baseline.
5. Flow view behavior remains unchanged.
6. This requirement supersedes REQ-FR-260038 item 4 for selected-state visibility semantics.

## Rationale

Separating selection-state signaling from initiation-affordance signaling reduces visual overload and clarifies interaction intent.

## Out of Scope

- Full visual redesign beyond Network-view interaction-state semantics.
- Changes to token governance baseline already defined by existing branding constraints.
