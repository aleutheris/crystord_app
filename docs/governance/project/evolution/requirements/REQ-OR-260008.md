# REQ-OR-260008: Search Filtering Delegates to Backend GraphQL Without Overfetch

- Class: OR
- Status: Accepted
- Date: 2026-05-10
- Source decision: Search data freshness and backend-filtering decision

## Requirement

For submitted workspace searches, the frontend shall delegate filtering to the backend GraphQL query that supports the active search contract, instead of fetching a broader dataset and filtering it locally in the browser.

## Rationale

The product direction favors accurate, current backend-backed results and controlled fetch scope over instant client-side re-filtering. Using GraphQL as the filtering boundary avoids unnecessary overfetch and keeps the search contract explicit.

## Scope Notes

- In scope for this requirement: submit-time GraphQL queries whose variables express the active label/text search intent.
- In scope for this requirement: treating the backend response as the source of truth for the submitted search.
- Out of scope for this requirement: autocomplete, predictive suggestions, and full-workspace client caches for ad hoc re-filtering.

## Verification

- Integration check confirms submitted searches issue a GraphQL request instead of reusing only local filtered state.
- Request-shape check confirms the active labels/text are sent as backend query variables.
- Runtime check confirms a backend data change can be observed on the next submitted search without a full page reload.
- Scope check confirms the frontend does not intentionally overfetch the workspace just to simulate search locally.
