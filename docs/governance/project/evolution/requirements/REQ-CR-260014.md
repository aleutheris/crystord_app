# REQ-CR-260014: View-Switching Must Not Change Shell and Panel Contracts

- Class: CR
- Status: Accepted
- Date: 2026-05-12
- Source decision: Dual-view rollout preserves existing workspace shell contracts

## Requirement

Introducing Network/Flow view switching shall not modify the top bar, side panels, or their interaction contracts. Only the center graph canvas representation may vary by selected view.

## Rationale

The change is intended as a representation enhancement, not a workspace information architecture change. Preserving shell and panel contracts reduces regression risk and keeps user orientation stable.

## Scope Notes

- In scope for this requirement: center-canvas representation swap, tabbed view selector, synchronized panel behavior.
- Out of scope for this requirement: navigation model redesign, side-panel layout changes, top-bar workflow changes.

## Verification

- UI check confirms top bar and sidebars are unchanged when switching views.
- Behavior check confirms panel interactions (selection details, search result focus) remain contract-compatible.
- Regression checks confirm no routing or shell-ownership boundary violations.
