# Functional Requirements: Explicit Atom Creation

## REQ-FR-260048: Explicit Atom Creation Trigger

The system shall provide a dedicated, visible UI control (e.g., a "Create Atom" button) on the graph canvas view for initiating the atom creation process. The previous implicit creation method (double-clicking the canvas) shall be disabled.

## REQ-FR-260049: Modal Creation Form

Activating the "Create Atom" control shall present the user with a modal data entry form for the new atom's properties. While this form is active, interaction with the rest of the application's UI (e.g., the graph canvas, search bar) shall be disabled.

## REQ-FR-260050: Form Consistency

The fields presented in the atom creation form must be identical to the fields available in the `AtomDetailPanel` used for editing an existing atom. This ensures a consistent data entry experience for both creating and updating atoms.

## REQ-FR-260051: Search-Centric Persistence

Upon successful submission of the creation form, the system shall persist the new atom to the backend and then immediately re-execute the current search query stored in the application's state. The graph canvas will refresh to show the results of this query.

## REQ-FR-260052: Post-Creation Feedback

The system must provide clear feedback to the user confirming the successful creation of the atom. This feedback (e.g., a temporary notification) is critical because the newly created atom may not be visible on the canvas if it does not match the active search criteria.

## REQ-FR-260053: Cancellation

The user shall be able to cancel the atom creation process from the modal form at any time, returning the application to its previous state without creating a new atom.
