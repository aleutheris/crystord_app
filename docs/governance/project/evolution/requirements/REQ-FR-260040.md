# REQ-FR-260040: Network View Magnetic Attach Must Activate Only on Target Atom Hit Area

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view connection drag target activation and hit-testing behavior
**Related ADRs**: ADR-260038
**Related Requirements**: REQ-FR-260038, REQ-FR-260035

## Summary

In Network view, magnetic attach during connection drag shall activate only when the pointer is on top of the target atom acceptance area, defined as the target atom disk plus outer initiation ring.

## Specification

1. Target magnetic attach must not activate from proximity outside the target acceptance area.
2. Target acceptance area includes full target atom disk and outer initiation ring.
3. Center-pointer overlap inside target atom disk is a valid attach/complete region.
4. Release over blank canvas remains no-op.
5. Release over source atom remains no-op.
6. Flow view behavior remains unchanged.

## Rationale

Strict hit-area activation avoids premature snapping and improves user confidence in intentional target selection.

## Out of Scope

- Changes to persisted bond model.
- Touch-specific gesture redesign beyond current desktop-first interaction scope.
