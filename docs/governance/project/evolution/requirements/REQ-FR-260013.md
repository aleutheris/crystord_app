# REQ-FR-260013: MVP Top-Level Areas and Shell Routing Structure

- Class: FR
- Status: Accepted
- Date: 2026-04-17
- Source decision: F1, F6 (Main areas and public/authenticated split)

## Requirement

The frontend shall implement MVP top-level areas for entry, workspace, and minimal placeholders, using a single application with route/layout boundaries that separate public-shell routes (demo sign-in and placeholders) from authenticated workspace-shell routes (graph/search/editing flows).

## Rationale

A clear shell structure keeps user entry predictable while protecting workspace flows behind explicit access boundaries.

## Scope Notes

- In scope for MVP: entry route, workspace routes, placeholder routes, public/authenticated layout split with route guards.
- Out of scope for this requirement: separate frontend deployments and production-grade identity federation.

## Verification

- Route checks confirm entry/workspace/placeholder areas exist and are reachable per shell rules.
- Access checks confirm workspace routes require successful demo sign-in.
- Layout checks confirm public-shell and workspace-shell UI boundaries are distinct.
