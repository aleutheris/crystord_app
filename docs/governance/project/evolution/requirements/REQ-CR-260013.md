# REQ-CR-260013: Autocomplete and Incremental Search Behaviors Are Deferred

- Class: CR
- Status: Accepted
- Date: 2026-05-10
- Source decision: Defer autocomplete and incremental search UX from the near-term model

## Requirement

The MVP and near-term search model shall not enable autocomplete, incremental backend suggestion fetching, or client-side live search re-scope while the user is typing.

## Rationale

The accepted near-term search model is intentionally simple: users compose search input, then explicitly submit. Deferring predictive and incremental behaviors reduces UX ambiguity and avoids introducing interaction paths that are not yet required.

## Scope Notes

- In scope for this requirement: disabling autocomplete UI, live suggestion menus, and typing-driven graph/result changes.
- In scope for this requirement: retaining internal extension seams for future search enhancements, as long as they are not enabled in current behavior.
- Out of scope for this requirement: future post-MVP autocomplete, local refinement after search submission, and other progressive search-assist patterns.

## Verification

- UI check confirms no autocomplete or suggestion list opens while typing.
- Runtime check confirms typing alone does not trigger graph/result updates.
- Code-path check confirms any retained local-filter or autocomplete structure is inactive in current behavior.
