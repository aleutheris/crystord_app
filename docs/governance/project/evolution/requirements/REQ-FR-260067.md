# Requirement: REQ-FR-260067 - Account Management and Identity Settings

- **Date**: 2026-06-23
- **Status**: Proposed
- **Type**: Functional Requirement

## Description

The system must provide an account-settings surface (a new `account-settings` feature module replacing
the admin placeholder) where a signed-in user can manage their identity using 8.1.0 operations:
view `me { username email emailVerified authMethods }`, `setPassword`, `unlinkAuthMethod`,
`requestEmailChange` → `confirmEmailChange`, `revokeAllSessions` ("sign out everywhere"), and
`deleteMyAccount`.

## Scenarios

- **Scenario 1: View account**
  - **Given** a signed-in user opens account settings,
  - **Then** the system shows their username, email, verification status, and linked auth methods from
    `me`.

- **Scenario 2: Set or replace password**
  - **When** the user submits a valid new password (REQ-CR-260023) via `setPassword`,
  - **Then** the system updates it and confirms.

- **Scenario 3: Remove an auth method**
  - **When** the user removes a method via `unlinkAuthMethod`,
  - **Then** the system blocks removing the **last** method (`AUTH-CANNOT-REMOVE-LAST-METHOD`) with an
    explanation.

- **Scenario 4: Change email**
  - **When** the user requests an email change and confirms the code sent to the **new** address,
  - **Then** the system updates the email; a taken target shows `EMAIL-ALREADY-IN-USE` inline.

- **Scenario 5: Sign out everywhere / delete account**
  - **When** the user chooses sign-out-everywhere (`revokeAllSessions`) or `deleteMyAccount`,
  - **Then** the system performs it; deletion is blocked with a clear reason while the user owns atoms
    or is a sole workspace Admin.
