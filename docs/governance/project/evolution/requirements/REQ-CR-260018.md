# REQ-CR-260018: Atom Keyboard Deletion Must Be Delete-Key Only

- Class: CR
- Status: Accepted
- Date: 2026-05-14
- Source decision: ADR-260015

## Requirement

Atom deletion triggered from keyboard interaction shall be bound to the Delete key only. Backspace shall not initiate atom deletion from canvas keyboard handlers.

## Rationale

Backspace is a common text-editing key and should not act as a destructive graph shortcut. Restricting deletion to Delete reduces accidental destructive intent while preserving explicit keyboard deletion support.

## Scope Notes

- In scope: canvas-level atom deletion shortcut mapping and guard conditions needed to prevent Backspace-triggered atom deletion.
- In scope: ensuring dialog/input text editing behavior remains unaffected by canvas delete shortcuts.
- Out of scope: changes to non-keyboard delete controls (contextual buttons, confirmation dialog buttons).

## Verification

- Keyboard behavior checks confirm Delete key can still open atom-delete confirmation when an atom is selected on canvas.
- Keyboard behavior checks confirm Backspace does not trigger atom-delete confirmation from canvas.
- Interaction checks confirm text inputs continue to use Backspace/Delete strictly for text editing while focused.
