# REQ-QR-260004: Dual-View Visual States Must Be Contrast-Safe and Non-Color-Only

- Class: QR
- Status: Active
- Date: 2026-05-12
- Source decision: visual state meaning must remain accessible and unambiguous across both views

## Requirement

Dual-view node/edge state representation shall satisfy practical WCAG 2.1 AA contrast expectations for key text/controls and shall not encode critical state meaning using color alone.

## Rationale

State comprehension must remain reliable for users with color-vision limitations and in varied display conditions.

## Scope Notes

- In scope for this requirement: contrast baseline for key controls/text, secondary non-color cues for selected/hover/connected/error states.
- Out of scope for this requirement: full formal accessibility certification beyond MVP baseline.

## Verification

- Visual check confirms non-color cue exists for core node/edge states.
- Contrast check confirms key text/control contrast is AA-aligned in dual-view contexts.
- Regression check confirms state semantics remain consistent between Network and Flow views.
