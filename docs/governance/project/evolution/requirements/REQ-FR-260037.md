# REQ-FR-260037: Network View Must Apply Topology-Aware Automatic Layout

**Status**: Accepted
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view initial layout, re-layout trigger, large-graph degrade policy, drag-position preservation
**Related ADRs**: ADR-260035
**Related Requirements**: REQ-FR-260030, REQ-FR-260031, REQ-QR-260003, REQ-FR-260035, REQ-FR-260036

## Summary

Network view shall apply a topology-aware force-directed layout on initial load and on fresh search results, so that connected atoms cluster naturally rather than appearing in an arbitrary grid. Users may re-trigger layout on demand, and manually dragged positions are preserved until layout is explicitly re-run.

## Specification

1. On first render of a search result set (no prior node positions), Network view shall compute a force-directed layout before initial paint.
2. Layout places connected atoms closer together and pushes unconnected atoms apart based on bond topology.
3. Manually dragged node positions are preserved across data refreshes using the existing `mergeNodePositions` pattern; force layout is not re-applied to nodes that already have stored positions.
4. A "Re-layout" action (toolbar button or equivalent) allows users to discard stored positions and recompute layout for the current graph.
5. Force simulation is run synchronously before render (iteration-capped) to avoid janky mid-paint animation on initial load.
6. Minimum node separation equals `NODE_SIZE + padding` to prevent overlap.
7. Above the reduced-mode threshold (> 150 nodes) and up to the blocked threshold (≤ 400 nodes), layout is computed in reduced mode; above 400 nodes, layout computation is skipped and grid fallback is used.
8. Self-loop edges are excluded from force simulation link calculations.

## Rationale

A fixed square grid ignores bond topology. Connected atoms landing far apart and unconnected atoms landing adjacent creates visual noise that undermines the graph's purpose of showing structural relationships clearly.

## Out of Scope

- Animated layout transitions between arrangements.
- Hierarchical / layered layout strategies (Dagre, ELK) — deferred as optional future strategy toggle.
- Reproducible deterministic layout across sessions (seeded PRNG) — deferred to future if needed.
- Layout for Flow view — unchanged.
