# Requirement: REQ-FR-260066 - Google Account Linking and Unlinked-Account Recovery

- **Date**: 2026-06-23
- **Status**: Proposed
- **Type**: Functional Requirement

## Description

The system must handle Google sign-in under 8.1.0's no-auto-link behavior. An unknown email creates and
signs in a new account (unchanged shape). An existing account that has not linked Google returns
`AUTH-GOOGLE-NOT-LINKED` and must be guided to sign in another way and then link Google from account
settings via `linkGoogle(idToken)`.

## Scenarios

- **Scenario 1: Unlinked existing account**
  - **Given** an email already has a non-Google account,
  - **When** the user attempts Google sign-in and the server returns `AUTH-GOOGLE-NOT-LINKED`,
  - **Then** the system shows inline recovery guidance ("sign in with your password, then link Google
    in settings") rather than a hard error.

- **Scenario 2: Link Google from settings**
  - **Given** a signed-in user in account settings,
  - **When** they choose "Link Google" and complete Google auth,
  - **Then** the system calls `linkGoogle(idToken)` and reflects the linked method in `me`.

- **Scenario 3: Email mismatch**
  - **When** `linkGoogle` returns `AUTH-GOOGLE-EMAIL-MISMATCH`,
  - **Then** the system explains the mismatch and does not link.
