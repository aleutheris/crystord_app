# Requirement: REQ-CR-260022 - Auth Entry Screen Must Use Two-Column Auth + Demo Panel Layout

- **ID**: REQ-CR-260022
- **Type**: CR (Constraint)
- **Status**: Accepted
- **ADR**: ADR-260053
- **Backlog Item**: BI-260052

## Statement

The `SignInPage` must present a two-column layout: a tabbed auth card (Sign In / Sign Up) in the
left column and a "Try Demo" feature card in the right column. On narrow viewports (≤640px) the
layout collapses to a single column.

## Rationale

The crystord_platform site establishes this layout as the product's visual identity for entry.
The web app must be consistent with that experience while using the web project's token system as
the authoritative brand source.

## Constraints

- Tab interface must follow ARIA tab pattern (role="tablist", role="tab", aria-selected, role="tabpanel").
- Demo card must display a feature list with checkmarks and a "No registration required" note.
- All colors must use web project CSS custom property tokens — no hardcoded hex values.
- Responsive breakpoint at 640px: layout stacks vertically.
