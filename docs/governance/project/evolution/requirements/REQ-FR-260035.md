# REQ-FR-260035: Network View Bonds Must Anchor at Geometrically Closest Circle Boundary Points

**Status**: Accepted
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view edge anchoring behavior, node-geometry edge routing, drag-created and fetched bond rendering
**Related ADRs**: ADR-260033, ADR-260029
**Related Requirements**: REQ-FR-260031, REQ-CR-260015, REQ-QR-260003

## Summary

In Network view, each rendered bond shall attach to each atom at the geometrically closest boundary point on the atom circle, rather than fixed-side attachment, to preserve readability on dense structures.

## Specification

1. For every rendered bond in Network view, source and target anchors are computed from current node geometry and bond endpoints.
2. Anchors represent the closest boundary points on source and target circles along the endpoint-to-endpoint vector.
3. Anchor computation is applied for both existing bonds loaded from backend and newly created bonds.
4. Anchors are recomputed when node geometry changes (drag, layout reflow, resize).
5. Zoom and pan do not change anchor semantics.
6. Flow view semantics remain unchanged.
7. Network-view bond labels may be omitted by default; edge label data remains preserved for optional future rendering.

## Rationale

Fixed-side attachment creates visually noisy crossings in dense graphs. Geometric closest-point anchoring improves spatial legibility while preserving the same data model and interaction model.

## Out of Scope

- Automatic layout algorithm selection.
- Arrowhead direction styling decisions.
- Changes to backend bond model.
- Mandatory always-on edge label rendering in Network view.
