# REQ-FR-260041: Network View Drag-Time Target Snap Must Use Nearest-Boundary Geometry

**Status**: Accepted
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view connection drag preview endpoint placement while target attach is active
**Related ADRs**: ADR-260038, ADR-260033
**Related Requirements**: REQ-FR-260035, REQ-FR-260039

## Summary

When target attach is active during connection drag in Network view, preview endpoint snapping shall follow nearest-boundary geometry instead of fixed-side snap points.

## Specification

1. Drag-time snap endpoint on active target uses nearest-boundary geometry along source-target direction.
2. Fixed four-side snap behavior is not the drag-preview baseline.
3. Established edges continue to use existing nearest-boundary anchoring rules.
4. Flow view behavior remains unchanged.

## Rationale

Geometry-consistent snapping reduces visual discontinuity between drag preview and final rendered edge anchoring.

## Out of Scope

- Redefinition of arrowhead semantics.
- Changes to backend graph contracts.
