# REQ-OR-260002: GraphQL Schema Stability and Versioning Contract

- Class: OR
- Status: Accepted
- Date: 2026-04-13
- Source decision: B3 (API compatibility rule)

## Requirement

The backend shall expose `Query.schemaInfo` with fields `schemaVersion` (semver), `schemaHash` (diagnostic hash), and `releasedAt` (RFC3339 timestamp) as the authoritative schema compatibility contract.

The frontend shall query `schemaInfo` once at startup, compare `schemaVersion` against a configured supported semver range, and fail fast with an explicit compatibility error when incompatible.

Deprecation signaling shall use dual channels: GraphQL `@deprecated` directives in schema and governance records in release notes, ICR artifacts, and a dedicated deprecation log.

## Rationale

Schema compatibility requires a deterministic machine-readable handshake and explicit governance visibility for migrations and removals.

## Scope Notes

- In scope: `Query.schemaInfo` contract, startup semver compatibility check, fail-fast behavior, dual-channel deprecation signaling.
- Out of scope: fallback feature flags for incompatible versions, alternative non-GraphQL metadata endpoints.

## Verification

- Contract test confirms `schemaInfo` returns `schemaVersion`, `schemaHash`, and `releasedAt` with expected formats.
- Startup validation test confirms semver-range check and explicit compatibility error on mismatch.
- Schema test confirms deprecated fields/arguments include GraphQL `@deprecated` directives.
- Governance test confirms release notes, ICR references, and deprecation-log entries exist for planned removals.
