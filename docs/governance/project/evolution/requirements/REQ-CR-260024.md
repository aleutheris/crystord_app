# Requirement REQ-CR-260024: Anti-Enumeration Authentication Messaging

- **Status**: Proposed
- **Description**: Auth flows MUST NOT reveal whether an email is registered. `beginSignup` always
  returns `true`, so its result MUST NOT be used to tell the user "email already exists"; password-reset
  request and rate-limit responses MUST use generic copy. Account existence may only be implied at the
  point where the user proves control (e.g. `SIGNUP-ACCOUNT-ALREADY-EXISTS` after code entry).
- **Rationale**: Prevents account-enumeration; aligns with the backend's intentional anti-enumeration
  design and the product's trust/ethics branding.
- **Related ADRs**: [ADR-260055](ADR-260055.md), [ADR-260058](ADR-260058.md)
