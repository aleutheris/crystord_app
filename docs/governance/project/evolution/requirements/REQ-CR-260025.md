# Requirement REQ-CR-260025: Centralized GraphQL Error-Code Interpretation

- **Status**: Proposed
- **Description**: All auth/authz error handling MUST go through a single `api-contract` error-code
  mapper that parses the code from the GraphQL `errors[].message` (not `extensions.code`, which 8.1.0
  does not populate for these codes) into a typed outcome. Features MUST NOT parse raw error messages
  themselves. The mapper MUST cover every code in the epic error table and provide a safe
  "unexpected error" fallback for unknown codes.
- **Rationale**: One tested translation layer gives consistent UX and removes the silent-expiry class of
  bug caused by per-feature, `extensions.code`-based handling.
- **Related ADRs**: [ADR-260057](ADR-260057.md), [ADR-260056](ADR-260056.md)
