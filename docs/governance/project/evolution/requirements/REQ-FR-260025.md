# REQ-FR-260025: Multi-Label Chip Input with Space/Colon Delimiters

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-search
**Affects**: SearchBar.tsx
**Related ADRs**: ADR-260023
**Related Requirements**: REQ-FR-260024

## Summary

Search input must support multi-label chip creation: user types a label name, presses Space or Colon (`:`) to convert it into a visual chip, and can repeat to stack multiple label chips before submission.

## Specification

### Chip Creation

1. User types a label name in the search input
2. User presses **Space** or **Colon** (`:`)
3. The typed text is converted to a visual chip (balloon) and inserted into the input
4. The input field clears and is ready for the next label entry
5. User can repeat steps 1–4 to add multiple label chips

### Chip Appearance

- Chips are rendered **inside the search input field** (not in a separate area)
- Chips appear before the cursor/text input area
- Each chip displays the label text and may include a visual affordance for removal

### Chip Removal

- User can remove the last chip by pressing **Backspace** when the input field is empty
- Backspace removes the trailing chip and does not clear the entire search

### Search Submission

- After chips are created and final text (if any) is entered, user presses **Enter** to submit
- Search query includes all chips (labels) and any remaining text in the input field

## Rationale

- **Multi-label support**: Balloons allow stacking of multiple labels in a compact, visual way
- **Delimiter choice**: Space and Colon are familiar delimiters in text-based UIs (e.g., email To fields)
- **In-field placement**: Keeps all search context visible and reduces visual clutter
- **Standard removal UX**: Backspace on empty field matches native list/chip removal patterns

## Out of Scope (Deferred)

- Label autocomplete while typing (see REQ-CR-260013)
- Drag-to-reorder chips
- Chip editing after creation

## Implementation Notes

- Override Space and Colon key handlers in search input
- Maintain chip state (array of labels)
- Render chips as visual elements within input container
- See BI-260023 for implementation task
