# REQ-FR-260031: Network View Circle Nodes and Connector-Based Edge Creation

**Status**: Superseded by REQ-FR-260038
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view node rendering, edge creation interaction, graph canvas pointer handling
**Related ADRs**: ADR-260029, ADR-260028
**Related Requirements**: REQ-FR-260030, REQ-FR-260009

Deprecated in favor of REQ-FR-260038 and ADR-260036, which replace side connector-dot initiation with full outer-circumference initiation while preserving circle nodes and boundary anchoring semantics.

## Summary

The Network view shall render atoms as circle nodes and shall provide a small side connector handle for initiating relationship creation. When the connector is dragged to another atom, the relationship is established and the edge anchors to the nearest boundary point on each circle.

## Specification

1. In the Network view, each atom is displayed as a circle.

2. Each circle exposes a small connector dot/handle on its side.

3. The user can drag from the connector dot to another atom to create a relationship.

4. On successful connection:
   - The relationship is established between both atoms.
   - The rendered edge anchors to the nearest edge point of each circle.

5. This behavior applies only to the Network view representation and does not alter Flow view rendering.

## Rationale

A Neo4j-like circle-and-connector model is a clear, familiar representation for graph relationships and gives the user an explicit affordance for starting an edge gesture.

## Out of Scope

- Alternative node shapes.
- Center-anchor edge creation only.
- Hover-only reveal of the connector handle.
- Changes to the underlying atom/bond data model.
