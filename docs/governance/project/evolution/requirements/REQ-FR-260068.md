# Requirement: REQ-FR-260068 - Authenticated Data and Taxonomy Reads

- **Date**: 2026-06-23
- **Status**: Proposed
- **Type**: Functional Requirement

## Description

Under 8.1.0 almost every read requires a valid Bearer session token, including reads that used to be
public (`retrieve`, `list_labels`, and all taxonomy reads). The public set is limited to `signin`,
`signinGoogle`, `schemaInfo`, `discoverOperations`, `beginSignup`, `completeSignup`,
`requestPasswordReset`, `confirmPasswordReset`, and `logout`. The client must attach the token to all
gated operations and must not issue any gated read before a token exists.

## Scenarios

- **Scenario 1: Token attached to reads**
  - **Given** a signed-in user,
  - **When** the app issues any data or taxonomy read,
  - **Then** the request carries `Authorization: Bearer <session-token>`.

- **Scenario 2: No gated read before authentication**
  - **Given** the app is starting,
  - **When** the startup sequence runs,
  - **Then** only `schemaInfo` (public) may run pre-auth; no workspace or taxonomy read runs before a
    token is present.

- **Scenario 3: Gated read without a token**
  - **When** a gated read returns `AUTHZ-AUTHENTICATION-REQUIRED`,
  - **Then** the system treats it as not-signed-in/expired and routes to sign-in (REQ-FR-260064).
