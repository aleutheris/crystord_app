# REQ-CR-260011: Branding Governance and Semantic Usage Constraint

- Class: CR
- Status: Accepted
- Date: 2026-04-17
- Source decision: ADR-260022

## Requirement

All UI and design-facing changes shall comply with branding semantics and constraints defined in `docs/governance/project/branding.md`, including token intent, state-color usage, and readability/accessibility-safe presentation.

## Rationale

Branding semantics are part of product trust and usability, not cosmetic preference. Treating branding as an explicit constraint keeps design behavior consistent across iterative delivery.

## Scope Notes

- In scope: UI copy tone, color semantics, state-color usage, token mapping, and visual constraints documented in branding policy.
- Out of scope: full visual redesign and future brand-system tooling automation.

## Verification

- Main-branch checklist confirms branding alignment for UI/design-facing changes.
- Governance traceability remains consistent across BI-260019, ADR-260022, and this requirement.
- Review checks verify that status communication does not rely on color alone and avoids forbidden branding patterns.
