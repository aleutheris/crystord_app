# REQ-OR-260001: MVP Demo Sign-In Operating Model

- Class: OR
- Status: Active
- Date: 2026-04-13
- Source decision: A6 (Product surfaces)

## Requirement

The MVP runtime shall require a lightweight demo sign-in step in development/testing using fixed credentials (`demo` / `demo`) before graph-workspace access, while avoiding full production authentication complexity.

## Rationale

Fast feedback on core graph behavior is operationally critical during MVP delivery.

## Scope Notes

- In scope for MVP: development/testing sign-in path with fixed demo credentials.
- Out of scope for this requirement: production multi-user authentication and authorization lifecycle.

## Verification

- Environment check confirms demo sign-in is available in development/testing runtime.
- Runtime check confirms successful demo login is required before workspace entry.
- Operational check confirms production configuration does not depend on demo credentials behavior.
