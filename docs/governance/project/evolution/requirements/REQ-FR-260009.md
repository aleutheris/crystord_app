# REQ-FR-260009: Canvas Visualization Modes and In-Workspace View Switching

- Class: FR
- Status: Accepted
- Date: 2026-04-17
- Source decision: D1, D2 (Visualization modes and view switching)

## Requirement

The frontend shall provide one primary graph canvas and support focused subgraph context through in-renderer filtering, with view/context switching performed inside the workspace shell via tabs/panels/context controls.

## Rationale

A single renderer with in-workspace switching preserves graph-first continuity and avoids route-level context loss.

## Scope Notes

- In scope for MVP: primary graph canvas, focused subgraph filtering, in-workspace tabs/panels for context switching.
- Out of scope for this requirement: multiple distinct graph-mode renderers and route-level workspace switches.

## Verification

- UI checks confirm graph and focused subgraph contexts are shown in the same renderer.
- Workflow checks confirm users switch context without leaving workspace shell.
- Regression checks confirm route transitions are not required for core graph tasks.
