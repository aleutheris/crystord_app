# REQ-FR-260043: Bond-Name Dialog Keyboard Input Must Not Trigger Canvas Deletion Flows

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: bond-name dialog text entry and keyboard event boundaries between modal input and canvas shortcuts
**Related ADRs**: ADR-260015, ADR-260031
**Related Requirements**: REQ-FR-260010, REQ-FR-260033, REQ-CR-260004

## Summary

When the bond-name dialog is open and focused, text-editing keys used inside its input (including Backspace and Delete) must be handled as text-editing actions and must not trigger atom or bond deletion flows from canvas keyboard handlers.

## Specification

1. While the bond-name dialog is open, keyboard events originating from the bond-name input are scoped to the dialog.
2. Pressing Backspace in the bond-name input removes text only and does not open atom-delete confirmation.
3. Pressing Delete in the bond-name input removes text only and does not open atom-delete confirmation.
4. Canvas-level delete shortcuts are suppressed while modal text input focus is active.
5. On dialog close, normal canvas keyboard behavior resumes.
6. Flow and Network views both obey this dialog keyboard-isolation rule.

## Rationale

Users must be able to edit bond names naturally without accidental destructive popups or cross-surface shortcut collisions.

## Out of Scope

- Redesign of dialog visuals or layout.
- New shortcut mappings unrelated to deletion and text entry conflict resolution.
