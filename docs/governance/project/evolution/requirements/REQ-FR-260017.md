# REQ-FR-260017: MVP Framework, Routing, and Runtime Stack Baseline

- Class: FR
- Status: Active
- Date: 2026-04-17
- Source decision: H1, H7, H9 (Framework criteria, runtime boundary, concrete stack)

## Requirement

The frontend shall implement the MVP runtime using React + TypeScript with React Router in a single application that separates public/auth/workspace flows via route/layout boundaries.

## Rationale

This baseline aligns framework capability with accepted architecture boundaries and the graph-first workspace model.

## Scope Notes

- In scope for MVP: single-app runtime with soft shell split, route guards, TypeScript-first implementation.
- Out of scope for this requirement: separately deployed public/auth/workspace frontends.

## Verification

- Route checks confirm public/auth/workspace boundaries within one app.
- Build checks confirm TypeScript-first compile and route-guard behavior.
- Regression checks confirm added routes preserve the soft split model.
