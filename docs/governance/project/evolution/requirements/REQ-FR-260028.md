# REQ-FR-260028: Auto-Chip Incomplete Text on Enter

**Status**: Accepted
**Type**: Functional Requirement
**Module**: workspace-search
**Affects**: SearchBar.tsx, use-search.ts
**Related ADRs**: ADR-260027, ADR-260023
**Related Requirements**: REQ-FR-260024, REQ-FR-260025

## Summary

When the user presses Enter while text exists in the search input but has not yet been converted to a chip, that text must be automatically converted to a chip and included in the search submission.

## Specification

1. When the user presses Enter in SearchBar with incomplete input text:
   - The input text is automatically converted to a chip.
   - The new chip is appended to the existing chip list.
   - Search submission proceeds with all accumulated chips (existing + new).

2. The text-to-chip conversion uses the same validation and formatting as manual Space/Colon chip creation.

3. If the input text is empty or whitespace-only, Enter key behavior remains unchanged (search with current chips only).

4. The conversion happens before search submission, so the backend receives the complete, finalized label list.

## Rationale

- Prevents accidental all-atom searches caused by forgetting to create a chip before pressing Enter.
- Makes user intent explicit by converting typed text into a visual chip artifact.
- Simplifies composition of multi-label searches: users can type multiple labels and press Enter after each one without manual chip-creation gestures.
- Aligns with common search UX patterns (e.g., email recipients, filter tags).

## Out of Scope

- Autocomplete or suggestion UI while typing.
- Validation of whether the label exists in the backend; let backend handle unknown labels.
- Dynamic chip creation using delimiters other than Space/Colon/Enter.
