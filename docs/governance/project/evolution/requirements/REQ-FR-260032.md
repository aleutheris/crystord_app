# REQ-FR-260032: Dual-View Shared Dataset and Cross-View Selection Consistency

**Status**: Accepted
**Type**: Functional Requirement
**Module**: workspace-graph, workspace-search
**Affects**: graph state orchestration, view switching, selection state management
**Related ADRs**: ADR-260030, ADR-260028
**Related Requirements**: REQ-FR-260030, REQ-FR-260031

## Summary

Network and Flow views shall render from one shared search dataset and shall keep selection synchronized across view switches.

## Specification

1. A submitted search updates one shared graph-result state source.
2. Both Network and Flow views render from this shared state source.
3. Selection is shared across views.
4. Selecting an atom in one view is reflected when switching to the other view.
5. View switching does not create separate per-view result datasets.

## Rationale

A shared-state model preserves user context and guarantees representation consistency while keeping implementation complexity manageable.

## Out of Scope

- Per-view independent result state.
- View-specific selection semantics.
