# Requirement REQ-OR-260017: Backend Schema Compatibility Pinned to ~8.1.0

- **Status**: Proposed
- **Description**: `public/config.json` `backendSchemaRange` MUST be set to `~8.1.0` so the startup
  `schemaInfo` handshake accepts `8.1.x` patch releases automatically and rejects `8.2.0`+ until a new
  ICR approves it. The schema-compatibility handshake (REQ-OR-260002 / ADR-260013) MUST gate normal API
  usage on this range.
- **Rationale**: Matches the "backend-facing contract pinned to an approved version" governance while
  allowing low-risk patch adoption; the prior `^6.0.0` range is incompatible with the 8.1.0 deployment.
- **Related ADRs**: [ADR-260054](ADR-260054.md), [ADR-260013](ADR-260013.md)
- **Related ICR**: [ICR-260001](ICR-260001.md)
