# REQ-FR-260038: Network View Outer-Circumference Edge Initiation and Hover Selection Affordance

**Status**: Active
**Type**: Functional Requirement
**Module**: workspace-graph
**Affects**: network view node interaction zones, edge initiation gestures, pointer-state affordances
**Related ADRs**: ADR-260036, ADR-260033
**Related Requirements**: REQ-FR-260035, REQ-FR-260036, REQ-FR-260033

## Summary

In Network view, each atom uses a full outer circumference ring as the relationship-initiation affordance while preserving inner-body node dragging. The same ring highlight semantics are reused for hover, selected, and valid drop-target states.

## Specification

1. Network-view atoms remain circles.
2. A full 360-degree outer ring is used as the relationship drag-initiation affordance.
3. The previous side connector-dot affordance is not used.
4. The outer ring is visible on hover and when the atom is selected.
5. Clicking the outer ring without dragging selects the atom.
6. Inner circle body remains draggable for node movement.
7. Relationship drag can start from any angle on the outer ring.
8. While a relationship drag is active, a valid target atom reuses the same outer-ring highlight semantics.
9. Releasing drag on blank canvas is a no-op.
10. Releasing drag on the source atom is a no-op (self-bonds are not allowed).
11. Releasing drag on another atom circle triggers the normal bond-creation flow.
12. Flow view behavior remains unchanged.

## Rationale

A full-ring gesture model improves discoverability and interaction consistency without adding a separate connector symbol. Separating inner-body drag from outer-ring edge initiation keeps node movement and bond creation both available while reducing visual clutter.

## Out of Scope

- Changes to atom/bond data contracts.
- New bond label policy.
- Flow-view interaction changes.
- Touch-specific gesture redesign beyond parity with existing pointer behavior.
