# REQ-QR-260003: Large-Graph Degrade Policy for Dual-View Rendering

- Class: QR
- Status: Active
- Date: 2026-05-12
- Source decision: establish explicit large-result degrade thresholds for predictable responsiveness

## Requirement

The dual-view graph experience shall apply a deterministic degrade policy based on result size:

- Up to 150 nodes: full interaction mode.
- 151-400 nodes: reduced visual complexity mode.
- Above 400 nodes: require explicit user confirmation before rendering.

## Rationale

Defined thresholds provide predictable responsiveness and prevent UI lock-ups while preserving user control over large renders.

## Scope Notes

- In scope for this requirement: threshold gating, reduced complexity mode, explicit confirmation for >400.
- Out of scope for this requirement: silent truncation and hidden data dropping.

## Verification

- Performance-mode check confirms threshold transitions at 150 and 400 boundaries.
- UX check confirms warning + explicit continue gate for >400 nodes.
- Reliability check confirms large renders do not block core shell interactions.
