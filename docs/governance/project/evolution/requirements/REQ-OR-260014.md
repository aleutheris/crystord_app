# Requirement: REQ-OR-260014 - Authentication Backend Contract

- **Date**: 2026-06-08
- **Status**: Deprecated
- **Deprecated**: 2026-06-23 — Superseded by REQ-OR-260016 (Schema 8.1.0 auth/authz contract, ADR-260054).
  The permanent-JWT assumption below no longer holds; tokens are now revocable sessions (REQ-FR-260064).
- **Type**: Organizational Requirement

## Description

The frontend application must interact with the backend for all authentication operations via the established GraphQL API. The backend is the single source of truth for user identity and session validity.

## Details

- The frontend must send credentials, Google `idTokens`, and demo requests to the appropriate GraphQL mutations or queries.
- The frontend must expect a JWT-compatible session token upon successful authentication.
- All subsequent authenticated API requests must include this token in the `Authorization` header.
