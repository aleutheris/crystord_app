# REQ-OR-260004: Build, Tooling, and Auth-Evolution Operational Criteria

- Class: OR
- Status: Accepted
- Date: 2026-04-17
- Source decision: H5, H8 (Build/tooling criteria and auth integration criteria)

## Requirement

The delivery workflow shall enforce fast-feedback tooling (Vite-based dev loop, lint/type checks, test gates), explicit environment separation for demo vs production behavior, and an auth-adapter evolution path that can replace MVP demo auth without route/layout rewrites.

## Rationale

Operational consistency and planned auth evolution reduce delivery risk and avoid architectural rework post-MVP.

## Scope Notes

- In scope for MVP: build/lint/type/test CI gates, environment isolation, auth-adapter seam.
- Out of scope for this requirement: full production identity provider integration in MVP.

## Verification

- CI checks confirm lint/type/test gates execute and block failures.
- Environment checks confirm demo credentials behavior is isolated from production settings.
- Architecture checks confirm auth integration seam supports adapter replacement.
