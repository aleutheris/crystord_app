# REQ-OR-260010: Dual-View Mutation Reflection Must Be Centralized in Shared Graph State

- Class: OR
- Status: Accepted
- Date: 2026-05-12
- Source decision: mutation reflection across views is centralized and shared

## Requirement

Atom and bond create/edit/delete operations shall update a shared graph state path once, and both views shall reflect the mutation from that shared updated state.

## Rationale

Centralizing mutation reflection avoids per-view divergence and prevents duplicated mutation logic.

## Scope Notes

- In scope for this requirement: shared mutation application path, shared re-render behavior for both views.
- Out of scope for this requirement: separate per-view mutation adapters and view-local mutation state.

## Verification

- Behavior check confirms create/edit/delete in either active view is visible when switching views.
- Code-path check confirms mutation update logic is centralized and reused.
- Regression check confirms no per-view mismatch after consecutive mutation operations.
