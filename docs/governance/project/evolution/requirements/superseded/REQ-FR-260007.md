# REQ-FR-260007: Graph-First Interaction with Search Bootstrap

- Class: FR
- Status: Superseded by REQ-FR-260022
- Date: 2026-04-17
- Source decision: C1 (Interaction style)

Deprecated in favor of `REQ-FR-260022`, which requires a blank workspace after sign-in until the user performs an explicit search.

## Requirement

The frontend shall provide a graph-first interaction model where workspace initialization performs a default search to preload an initial atom/bond context, and users can refine the visible context via search-by-label controls.

## Rationale

Graph-first interaction requires immediate data context to avoid an empty-canvas start that blocks meaningful manipulation.

## Scope Notes

- In scope for MVP: default search bootstrap on workspace load, visible graph context, search refinement by label.
- Out of scope for this requirement: advanced filter builders, query history, saved searches.

## Verification

- Runtime check confirms workspace load triggers a default search request.
- End-to-end flow confirms loaded results appear in the graph canvas without manual first action.
- End-to-end flow confirms users can refine current graph context with label search.
