# Requirement REQ-OR-260013: Temporary Authentication Compatibility

- **Status**: Superseded by REQ-FR-260063, REQ-OR-260016
- **Deprecated**: 2026-06-23 — Superseded by REQ-FR-260063 and REQ-OR-260016 (ADR-260055). Schema 8.1.0
  collects a real `username` at `completeSignup`, so the email-as-username shim is retired.
- **Description**: For the initial implementation, the frontend MUST send the value from the username/email input field as the `email` parameter to the backend authentication service.
- **Rationale**: This provides a temporary bridge to support username/password login before the backend officially distinguishes between username and email fields, enabling faster delivery of the feature for use cases like the 'demo' account.
- **Related ADRs**: [ADR-260049](ADR-260049.md)
