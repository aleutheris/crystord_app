# REQ-CR-260017: Network View Ring Visual Semantics Must Use Distinct Tokenized Colors

- Class: CR
- Status: Active
- Date: 2026-05-14
- Source decision: ADR-260037

## Requirement

For Network view circle nodes, the outer connection-initiation ring and the inner selection visual must use distinct, centrally defined design tokens (no feature-local hardcoded color literals) so interaction semantics are clear and future branding updates are low-risk.

## Rationale

Distinct ring semantics improve affordance clarity (connection initiation vs selection state). Centralized token usage reduces branding-change cost and prevents color drift.

## Scope Notes

- In scope: outer ring color token, inner selection visual color token, tokenized usage in node rendering code, and tests validating token usage.
- Out of scope: full page branding redesign and non-network-view visual restyling.

## Verification

- Unit/architecture checks confirm outer-ring and inner-selection visuals resolve from centralized tokens.
- Review checks confirm no hardcoded ring/selection color literals remain in Network-view node interaction surfaces.
- Regression checks confirm visual distinction remains present in selected and hover states.
