# REQ-QR-260002: MVP Accessibility and Reliability Quality Baseline

- Class: QR
- Status: Active
- Date: 2026-04-17
- Source decision: I2, I4 (Accessibility baseline and reliability expectations)

## Requirement

MVP quality baseline shall enforce WCAG 2.1 AA practical coverage on core interaction paths and prohibit release when unacceptable reliability failures are present, including unrecoverable crashes, silent mutation data loss, broken route guards, frozen graph interactions, or non-actionable blocking errors.

## Rationale

Accessibility and reliability are minimum trust requirements for production-facing workflows.

## Scope Notes

- In scope for MVP: core-path accessibility checks and reliability failure blocking policy.
- Out of scope for this requirement: complete product-wide accessibility certification beyond MVP scope.

## Verification

- Accessibility checks confirm keyboard/focus/label/contrast compliance on primary routes.
- Reliability checks confirm no blocked failure categories remain at release gate.
- Release governance confirms blocked categories are treated as release-stopping defects.
