# Epic: EPIC-260067 — Classify Inspector: label & category chip editors

- Epic ID: EPIC-260067
- Title: Classify Inspector — label & category chip editors on the atom
- Persona/Actor: Knowledge worker classifying atoms.
- Value statement: For users classifying atoms, deliver first-class label + category editing in the Inspector so classification is fast and structured — killing the comma-separated text box.
- Context: The right panel (`DetailPanel.tsx`) edits labels as a comma-separated string — the weakest spot in the UI — and categories are invisible on the atom.
- Problem statement: There is no real label/category editing surface; the v9.2 hierarchical category model is unused on the atom.
- Expected outcome: An Inspector **Classify** tab with (a) a **label chip editor** — autocomplete from `listLabels`, deterministic desaturated brand-palette color, text always shown; and (b) **category facet chips grouped by dimension** (e.g. `Region ▸ Belgium`), assignment by most-specific `valueKey` with derived ancestors, breadcrumb path read-only, editing opens the shared category tree in "pick" mode.
- Success metric(s): Users add/remove labels and assign/clear categories on an atom via chips; the comma-box is gone; labels and categories are visually distinct.
- Non-goals: The left-rail category navigator (EPIC-260068); bulk classify board (EPIC-260075); taxonomy authoring/permissions.
- Constraints: Branding (REQ-CR-260011) — desaturated palette, never color alone. Labels (informal/flat/fast) must look distinct from categories (formal/hierarchical/structured).
- Dependencies: EPIC-260066 (inspector-tab registry; shared label chip editor + permission-aware category tree).
- Risks/assumptions: Assumes the shared category tree's "pick" mode (from EPIC-260066) is reusable here.
- Target window: TBD (Ideation).
- Validation approach: Unit tests for chip add/remove + category assignment; E2E editing labels/categories on an atom and persisting.

## Ideation Bullets

- [ ] Label chip color hashing within the restrained brand palette (deterministic, muted, same label = same color everywhere).
- [ ] Category facet-chip visual grammar (dimension-tagged) and how it reads distinctly from label pills.
- [ ] Breadcrumb path display (read-only) for most-specific value + derived ancestors.
- [ ] "Pick" mode reuse of the shared category tree from EPIC-260066.
- [ ] Persistence path through `change`/AtomInput categories.
- [ ] Full code + test coverage task.
