# Backlog Item: BI-260063 - Complete the Auth Sign-In/Sign-Up Tab ARIA Pattern

- **Date**: 2026-06-24
- **Status**: Done
- **Slice**: Accessibility polish (pre-existing; surfaced during the BI-260055 review)
- **Ready since**: 2026-06-25 — independent polish on `SignInPage.tsx`; no dependencies, no unknowns
  (the dual-view `GraphViewTabs` is the established WAI-ARIA precedent to mirror).
- **Done**: 2026-06-25

## Description

The Sign In / Sign Up tabs on the auth entry screen use `role="tab"` / `role="tabpanel"` /
`aria-selected`, but do not implement the full WAI-ARIA tab pattern: they lack `aria-controls` /
`aria-labelledby` links between tab and panel, roving `tabIndex`, and ArrowLeft/ArrowRight keyboard
navigation with focus management. This predates the verify-first work (it came from BI-260052 /
ADR-260053) and was only noted — not changed — while `SignInPage` was edited in BI-260055. The
dual-view *graph* tabs already follow the full pattern (REQ-CR-260016); the auth tabs should match.

## Goals

- Add `id` + `aria-controls` on each tab and `aria-labelledby` on the tabpanel.
- Roving `tabIndex` (selected tab `0`, others `-1`) and ArrowLeft/ArrowRight to move between tabs with
  focus following selection.
- Preserve the existing visible focus and tab labels/behavior.

## Acceptance Criteria

- The auth tablist passes the WAI-ARIA tab pattern checks (controls/labelledby wired, arrow-key nav,
  roving tabindex), consistent with the dual-view tabs.
- No change to the sign-in / sign-up / demo behaviors.

## Non-goals

- Any auth flow logic (sign-in, sign-up, demo, Google) — untouched.

## Dependencies

- None. Independent polish on `SignInPage.tsx`.

- Quality gates: `npm run lint`, `npm run typecheck`, `npm run test`.

## Done Evidence Block

- **`SignInPage.tsx` — full WAI-ARIA tab pattern.** The Sign In / Sign Up tablist now mirrors the
  dual-view `GraphViewTabs` precedent:
  - each tab has `id="auth-tab-{signin|signup}"` + `aria-controls="auth-tabpanel"`; the panel has
    `id="auth-tabpanel"` + `aria-labelledby` pointing at the selected tab;
  - roving `tabIndex` (selected `0`, other `-1`) so the tablist is a single tab stop;
  - `ArrowLeft`/`ArrowRight` move (wrapping) between tabs via an `onKeyDown` on the tablist, with focus
    following selection (tab `ref`s → `.focus()`); non-arrow keys are ignored.
  - The tab order/labels are a small `AUTH_TABS` constant; the markup is mapped over it.
- **No behavior change** to sign-in / sign-up / demo / Google / reset — only the tab affordance ARIA and
  keyboard nav were added (the existing click-to-switch and `aria-selected` are preserved).
- **Tests.** Added a "WAI-ARIA tab pattern" block to `SignInPage.test.tsx` (aria-controls/labelledby
  wiring, roving tabindex, ArrowRight select+focus+panel-relabel, ArrowLeft wrap, last→first wrap,
  non-arrow ignored). Strengthened the `architecture.test.ts` auth-tab assertion to require the full
  pattern (controls/labelledby, roving tabindex, arrow-key nav).

- **Commands.** `npm run lint` → 0; `npm run typecheck` → 0; `npm run test` → 0 (903 passing, 53 files);
  `npm run test:e2e` → 47 passed.
- **Rollback.** Revert `SignInPage.tsx` (and the strengthened arch assertion) — the tabs return to the
