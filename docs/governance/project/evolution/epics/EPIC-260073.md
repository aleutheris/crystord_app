# Epic: EPIC-260073 — History Inspector tab: field-level change audit

- Epic ID: EPIC-260073
- Title: History Inspector tab — field-level change audit
- Persona/Actor: Any user (and auditors) needing to see how an atom changed.
- Value statement: For users needing transparency, deliver a per-atom change history so they can see who changed what, when, and with what remark.
- Context: The backend records field-level change history (`shellies.changes`) — a branding pillar ("Transparency") — that is completely invisible in the UI.
- Problem statement: There is no audit/history surface; users cannot see an atom's change trail.
- Expected outcome: An Inspector **History** tab listing field-level changes for the selected atom (who · when · field · remark), in reverse-chronological order.
- Success metric(s): Users open an atom and read its change log.
- Non-goals: Sharing/grants (EPIC-260074); account-level audit; live collaboration (excluded by REQ-CR-260010 — this is async audit, not real-time presence).
- Constraints: Async audit only (no RTC); read-only.
- Dependencies: EPIC-260066 (inspector-tab registry).
- Risks/assumptions: Assumes `shellies.changes` exposes the fields/remarks needed for a readable log.
- Target window: TBD (Ideation).
- Validation approach: Unit tests for the change-log rendering; E2E viewing history after editing an atom.

## Ideation Bullets

- [ ] Change-log display + which fields/remarks come from `shellies.changes`.
- [ ] Ordering + pagination for long histories.
- [ ] Full code + test coverage task.
