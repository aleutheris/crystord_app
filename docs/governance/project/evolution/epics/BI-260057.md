# Backlog Item: BI-260057 - Self-Service Password Reset

- **Date**: 2026-06-23
- **Status**: Done
- **Slice**: S1 (unblocker)

## Description

Add the signed-out forgot/reset password flow so locked-out users can recover, using the public
`requestPasswordReset` ‚Üí `confirmPasswordReset` operations. A successful reset revokes all sessions, so
the user is returned to sign-in.

## Goals

- "Forgot password" entry on the sign-in page ‚Üí email ‚Üí `requestPasswordReset` with neutral
  anti-enumeration confirmation copy.
- Reset-confirmation screen: token + new password (validated via the shared schema) ‚Üí `confirmPasswordReset`.
- On success route to sign-in; handle `RESET-INVALID-OR-EXPIRED-TOKEN` by restarting the flow.

## Acceptance Criteria

- Requesting a reset never reveals whether the email exists (REQ-CR-260024).
- A valid token + compliant new password completes and routes to sign-in.
- Bad/expired token prompts a restart.

## Non-goals

- Email change and other account-settings flows (BI-260058+).

## Dependencies

- BI-260054 (auth documents, error mapper, validation schema).

- E2E: request ‚Üí confirm ‚Üí sign-in; invalid token restart.
- Quality gates: lint, typecheck, test, test:e2e.
- **Coverage: 100% line + branch on the reset modules (REQ-QR-260008).**
