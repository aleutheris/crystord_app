# Epic: EPIC-260068 — Categories Navigator: left-rail faceted tree

- Epic ID: EPIC-260068
- Title: Categories Navigator — left-rail faceted category tree
- Persona/Actor: Knowledge worker browsing and scoping by structure.
- Value statement: For users exploring what data exists, deliver a left-rail Categories lens (faceted tree) so they can see how data is organized and scope the working set by category.
- Context: The brand promise is "the user can see what data exists." Hierarchical categories (dimensions→values) have no browse surface today.
- Problem statement: The v9.2 category model is invisible; categories can't be browsed or used as filters; a hierarchy can't live in a text box.
- Expected outcome: A left-rail **Categories** lens = expanding facet tree via `retrieveCategoryBrowse` (children + per-value `atomCount` + atoms under a node). Faceted semantics map 1:1 to standard search: OR within a dimension, AND across dimensions, `includeDescendants` as one checkbox. Selecting values pushes chips to the top query bar (using the shared filter builder). Inline edit affordances everywhere — ＋ add dimension/value here, rename, drag-to-reparent (Q2).
- Success metric(s): Users expand categories with count badges and filter the working set; users author taxonomy inline without an admin surface.
- Non-goals: Per-atom category chips (EPIC-260067); the bulk-classify board (EPIC-260075); shipping taxonomy ownership/permissions (future — but build against the model).
- Constraints: Q2 — no admin-only management surface; build the tree and its edit controls against the same permission model as atom sharing, so they become permission-gated when that lands (no rework).
- Dependencies: EPIC-260066 (navigator registry; shared permission-aware category tree; filter builder).
- Risks/assumptions: Assumes `retrieveCategoryBrowse` returns the children/atomCount/atoms needed for folder-style expansion (per schema).
- Target window: TBD (Ideation).
- Validation approach: Unit tests for facet selection semantics (OR/AND/includeDescendants); E2E browsing + filtering + inline taxonomy edit.
