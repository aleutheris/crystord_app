# Backlog Item: BI-260056 - Session-Aware Auth State and Server-Side Logout

- **Date**: 2026-06-23
- **Status**: Done
- **Slice**: S1 (unblocker)

## Description

Make the client treat the token as a revocable, expiring session and recover gracefully: lazy re-auth
on expiry, a server-side logout, demo/URL tokens handled as sessions, and sign-in-page handling for
Google-not-linked and rate-limit responses.

## Goals

- Route any `AUTHZ-AUTHENTICATION-REQUIRED` (via the mapper) to sign-in by clearing local auth state;
  keep `localStorage` persistence with lazy re-auth (no blocking boot probe).
- `signOut()` calls `mutation { logout }` fire-and-forget, then clears local state regardless of result;
  add the Logout control to the shell header/account menu.
- Treat the demo session and the `?token=` handoff token as ordinary sessions (demo stays non-persisted).
- On the sign-in page, show inline recovery for `AUTH-GOOGLE-NOT-LINKED` and the generic
  `AUTH-RATE-LIMITED` message with submit back-off.

## Acceptance Criteria

- An expired/revoked session deterministically routes to sign-in with no raw error surfaced.
- Logout ends the server session and clears local state even if the network call fails.
- Demo and URL-handoff sessions follow the same re-auth behavior.
- Google-not-linked and rate-limit messages render per REQ-FR-260066 / REQ-CR-260026.

## Non-goals

- The "Link Google" settings action and `revokeAllSessions` UI (BI-260059/060).

## Dependencies
