# Epic: EPIC-260070 — Account & Settings: avatar menu & settings surface

- Epic ID: EPIC-260070
- Title: Account & Settings — top-right avatar menu and settings surface
- Persona/Actor: Any signed-in user managing their own account and environment.
- Value statement: For all users, deliver an account/settings surface so they can manage identity, workspace, and preferences — table stakes for real (non-demo) users.
- Context: The backend exposes a full account/identity layer (`setPassword`, `requestEmailChange`/`confirmEmailChange`, `authMethods`/`linkGoogle`/`unlinkAuthMethod`, workspace membership/roles) with no UI home. The top-right is three loose buttons (`ThemeToggle`, Account, Sign Out).
- Problem statement: No place to manage password/email/auth-methods/workspace/preferences; the Q3 compute-emphasis switch has no physical home.
- Expected outcome: A top-right **avatar dropdown** (consolidating today's `ThemeToggle`/Account/Sign-Out; **Sign Out pinned at the bottom**, separated) → a Settings view with: **Account & identity** (username, email via `requestEmailChange`/`confirmEmailChange`, `setPassword`, linked methods `authMethods`/`linkGoogle`/`unlinkAuthMethod`); **Workspace** (current workspace, switch, manage members & roles for owners/admins); **Preferences** (display density / theme + the Q3 compute-emphasis switch).
- Success metric(s): Users change password/email/auth-methods, switch workspace, and toggle compute emphasis from one menu; Sign Out is at the bottom of the dropdown.
- Non-goals: Per-atom sharing (EPIC-260074); taxonomy permissions (future).
- Constraints: The account-level complement to EPIC-260074's per-atom sharing; reuse the existing `AccountSettingsPanel`; preferences write the EPIC-260066 preferences contract.
- Dependencies: EPIC-260066 (top-bar account-menu slot, settings routes, preferences contract).
- Risks/assumptions: The avatar-dropdown slice is small/low-risk (reshapes existing controls, no new backend calls) and can be pulled early.
- Target window: TBD (Ideation).
- Validation approach: Unit tests for the dropdown + settings panes; E2E for password/email change and workspace switch.
