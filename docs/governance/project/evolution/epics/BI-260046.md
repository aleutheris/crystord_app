# BI-260046: Implement Explicit Atom Creation via Modal Sidebar

- **Status**: Ready
- **Date**: 2026-06-05
- **Effort**: Medium
- **Related Requirements**: `REQ-FR-260048`, `REQ-FR-260049`, `REQ-FR-260050`, `REQ-FR-260051`, `REQ-FR-260052`, `REQ-FR-260053`
- **Related ADRs**: `ADR-260046`

## Description

This backlog item covers the implementation of a new, explicit workflow for creating atoms. The current implicit double-click-to-create behavior will be replaced with a dedicated "Create Atom" button that uses a modal version of the existing `AtomDetailPanel` for data entry. This change ensures a more discoverable, consistent, and predictable user experience that respects the application's search-centric design.

## Acceptance Criteria

- A "Create Atom" floating button is present on both the Flow and Network graph views.
- Clicking the button opens the `AtomDetailPanel` and disables interaction with the rest of the UI (e.g., via a modal overlay).
- The panel's title in this mode is clearly "Create New Atom".
- The form fields are identical to the editing-mode panel.
- Clicking "Save" creates the atom, closes the panel, re-runs the current search, and displays a success notification.
- The newly created atom only appears on the canvas if it matches the current search.
- Clicking "Cancel" or a close button on the panel aborts the creation process.
- The old double-click-to-create functionality is completely removed.
- The entire feature is covered by comprehensive unit, integration, and end-to-end tests, aiming for 100% coverage of the new logic.

## Sub-Tasks

1.  **State Management**: Add `isCreatingAtom` state to `WorkspaceShell`.
2.  **UI - Button**: Implement the "Create Atom" button and place it in the canvas UI.
3.  **UI - Modal Overlay**: Create and conditionally render a backdrop component in `WorkspaceShell`.
4.  **Refactor `DetailPanel`**:
    - Add `isCreationMode?: boolean` and `onCreate?: (data) => void` props.
    - Conditionally change the title and save button behavior.
5.  **Refactor `WorkspaceShell`**:
    - Wire the "Create Atom" button to set `isCreatingAtom(true)`.
    - Conditionally render `DetailPanel` in creation mode, passing an `onCreate` handler.
    - Implement the `onCreate` logic: call mutation, set `isCreatingAtom(false)`, trigger search refetch, and show notification.
6.  **Cleanup**: Remove `onDoubleClick` handlers from `GraphCanvas` and `NetworkCanvas`.
7.  **Unit Tests**:
    - Write tests for the new state logic in `WorkspaceShell`.
    - Write tests for the `DetailPanel`'s creation mode.
    - Write tests for the new button and overlay.
8.  **E2E Test**:
    - Create a new Playwright test file (`e2e/create-atom.spec.ts`).
    - The test will:
        - Click the "Create Atom" button.
        - Verify the modal sidebar appears and the canvas is disabled.
        - Fill out the form and save.
        - Verify the success notification appears.
        - Verify the search is re-run and the canvas updates correctly (testing both a matching and non-matching creation).
