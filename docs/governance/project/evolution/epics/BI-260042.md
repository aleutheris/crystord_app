# BI-260042: Resolve Keyboard Deletion Conflicts Between Bond Naming and Canvas Shortcuts

**Status**: Done
**Type**: Governance + Delivery
**Module**: workspace-graph
**Component**: bond-name dialog keyboard handling and canvas atom-delete shortcut policy
**Blocks**: post-BI-260041 interaction hardening

## Dependencies

- ADR-260015: Visualization and Interaction Model for MVP
- ADR-260031: Dual-View Accessibility and Usability Baseline
- REQ-FR-260043: Bond-Name Dialog Keyboard Input Must Not Trigger Canvas Deletion Flows
- REQ-CR-260018: Atom Keyboard Deletion Must Be Delete-Key Only
- REQ-CR-260004: MVP Deletion Safety Constraints for Graph Editing

## Acceptance Criteria

1. While bond-name dialog input is focused, Backspace/Delete edit text only and never trigger atom-delete confirmation.
2. Canvas atom deletion via keyboard is available on Delete key only.
3. Backspace on canvas does not trigger atom-delete confirmation.
4. Existing delete-confirmation and undo safety flows remain intact.
5. Behavior is consistent in both Flow and Network views.
6. Tests validate:
   - Dialog input keyboard isolation from canvas shortcuts.
   - Delete-only atom keyboard deletion behavior.
   - No regression in existing delete confirmation and undo behavior.

## Verification

- Unit/integration tests cover dialog keydown behavior and canvas keyboard deletion guards.
- Regression checks confirm deletion safety behavior remains confirmation-first with undo support.

## Implementation Notes

- Updated keyboard deletion policy in `use-canvas-interactions.ts`:
   - Atom/edge deletion shortcut now triggers on Delete key only.
   - Backspace no longer triggers canvas deletion flow.
   - Added editable-target guard so key events from input/textarea/select/contenteditable elements do not invoke canvas deletion handlers.
- Added `use-canvas-interactions.test.tsx` with focused coverage for:
   - Delete key opens confirmation for selected atom.
   - Backspace does not open deletion confirmation.
   - Delete in an input element does not trigger canvas deletion.
- Updated architecture assertion in `architecture.test.ts` to enforce Delete-only shortcut expectation.
