## Implementation Decision Locks (Approved)

Approved on: 2026-04-17

- Feature boundaries and public APIs:
  - Each feature exposes integration points only through its public API/barrel.
  - Cross-feature integration must consume public APIs only.
  - `api-contract` is the single frontend GraphQL boundary.
- State partition ownership:
  - Server state is accessed through api-contract/query adapters.
  - Shared workspace state is owned by workspace shell/context.
  - Feature-local state is owned/mutated inside the feature boundary.
  - Transient UI state remains local and short-lived.
- CSR-first runtime lock:
  - Interactive workspace routes are CSR-first.
  - No SSR-first dependency for graph/search/detail workflows in MVP.
  - Any future SSR usage is explicitly out of MVP workspace scope.
- Dependency enforcement:
  - Forbidden cross-feature internal imports must fail checks.
  - Dependency enforcement is CI-blocking (not warning-only).
  - Shared adapters require explicit allowlist/governance.
- Contract stability gates:
  - Breaking changes to feature public interfaces require explicit approval evidence.
  - Contract tests are required for cross-feature payloads/adapters.
  - Review workflow must enforce interface stability controls.
- Non-goal hard constraints:
  - No micro-frontend split in MVP.
  - No SSR-first workspace rendering in MVP.
  - No unmanaged cross-feature internal coupling.
- Verification gates:
  - Architecture/dependency checks pass with boundary enforcement.
  - Contract tests for defined seams pass.
  - CSR-first workspace behavior is validated.
  - Backlog Done note includes requirement-linked evidence mapping.
