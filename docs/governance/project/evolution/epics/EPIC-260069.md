# Epic: EPIC-260069 — Compute: formula builder & Flow visualization

- Epic ID: EPIC-260069
- Title: Compute — formula builder, computation transparency & Flow visualization
- Persona/Actor: Analyst-style user (heavy math / derived values); also any user with computed atoms.
- Value statement: For analyst-style users, deliver a formula builder plus computation transparency so derived values are easy to author and self-explaining — the product's headline differentiator (Q3).
- Context: The backend `operation` engine (SUM/MINUS/PRODUCT/DIVIDE/COLLECT, typing, dependency graph) exists; today the UI can't even edit the field. Highest-ceiling area, currently a void.
- Problem statement: No way to create or inspect computed atoms; the compute engine and its dependency graph are invisible.
- Expected outcome: An Inspector **Compute** tab = a formula builder — function picker from `discoverOperations`; typed argument slots (each arg is an atom reference picked by title via search, or a constant key; COLLECT's filter slot reuses the shared filter builder, label-only until backend adds category collect). Manual-vs-computed is a hard fork: once an atom has an `operation`, content flips to read-only "result" mode. An **"Explain this value"** popover surfaces `evaluationStatus`/`errorCode`/`causes`/`cycleNodes/Edges` as plain language. Computation is visualized in the **Flow** view: `OP_DEPENDENCY` edges as a distinct "computed-from" style, status badges on nodes (via the EPIC-260066 node seam), cycles highlighted.
- Success metric(s): Users build a SUM/COLLECT atom without typing JSON; computed atoms show up-to-date/error status + an explanation; dependencies render in Flow.
- Non-goals: Charts/analytics (excluded by REQ-CR-260010 — see constraint); category-based COLLECT (backend not built — starts label-only).
- Constraints: Realized with the **current** stack (React Flow + DOM), **no charting library** (REQ-CR-260010); reads the Q3 emphasis flag (default from EPIC-260066; the toggle UI is EPIC-260070).
- Dependencies: EPIC-260066 (inspector-tab + view registries, shared filter builder, node-extension seam).
- Risks/assumptions: Atom-reference picker is single-consumer and lives in this epic. Assumes `discoverOperations` + the evaluation fields are available per schema.
- Target window: TBD (Ideation).
- Validation approach: Unit tests for the formula builder + "Explain this value" states; E2E creating a computed atom and seeing status + dependency edges.
