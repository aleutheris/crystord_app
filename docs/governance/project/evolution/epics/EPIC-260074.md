# Epic: EPIC-260074 — Share Inspector tab: atom-level access grants

- Epic ID: EPIC-260074
- Title: Share Inspector tab — atom-level access grants
- Persona/Actor: Any user controlling who can see/edit their atoms.
- Value statement: For users controlling access, deliver per-atom sharing so they can grant and revoke who can view or edit an atom.
- Context: The backend supports atom-level sharing (`shareAtom` / grants / roles, `revokeAtomAccess`) — a branding pillar ("Control") — invisible in the UI. Read-side access gating already exists (BI-260061).
- Problem statement: There is no sharing/access surface; users cannot manage who has access to an atom.
- Expected outcome: An Inspector **Share** tab — view current grants and grant/revoke access by principal (user/workspace), principal type, and access level (`shareAtom` / `revokeAtomAccess`).
- Success metric(s): Users share an atom with a principal at a chosen level and see/revoke who has access.
- Non-goals: Account/workspace settings (EPIC-260070); taxonomy permissions (future extension that will mirror this model, per Q2); live collaboration (async only).
- Constraints: Async sharing (no RTC, per REQ-CR-260010); consistent with the existing access-control gating (BI-260061).
- Dependencies: EPIC-260066 (inspector-tab registry).
- Risks/assumptions: The same grant/role mechanism is the planned basis for taxonomy permissions (Q2) — design with that reuse in mind.
- Target window: TBD (Ideation).
- Validation approach: Unit tests for grant/revoke flows + access-level display; E2E sharing an atom and confirming the grant.
