# Backlog Item: BI-260054 - API-Contract Foundation for Schema 8.1.0

- **Date**: 2026-06-23
- **Status**: Done
- **Slice**: S1 (unblocker)

## Description

Lay the `api-contract` foundation for the 8.1.0 auth/authz migration: pin the schema range, add the new
auth operation documents, introduce the central GraphQL error-code mapper, rewrite session-expiry
detection to read codes from the GraphQL `message`, and verify no gated read runs before a token exists.
Every other epic slice depends on this item.

## Goals

- Bump `public/config.json` `backendSchemaRange` `^6.0.0` → `~8.1.0`; keep the `schemaInfo` startup gate.
- Add GraphQL documents for `beginSignup`, `completeSignup`, `logout`, `revokeAllSessions`,
  `requestPasswordReset`, `confirmPasswordReset`, `requestEmailChange`, `confirmEmailChange`, `me`,
  `setPassword`, `unlinkAuthMethod`, `linkGoogle`, `deleteMyAccount` (consumed by later slices).
- Add a single error-code mapper: `errors[].message` → typed code enum → user-facing outcome, covering
  every code in the epic error table plus an unknown-code fallback.
- Rewrite Apollo error handling / `session-expired` to detect `AUTHZ-AUTHENTICATION-REQUIRED` from the
  message (keep the 401 fallback).
- Verify Bearer attaches to all gated operations and that startup issues no gated read pre-auth.
- Extend the `retrieve` document and atom adapter/types to request `ownerUuid` and the caller's
  `accessLevel`, so the S3 read-side gating slice (BI-260061) has the data it needs. (Only the field
  exposure lands here; the gating logic stays in BI-260061.)

## Acceptance Criteria

- `schemaInfo.schemaVersion` outside `~8.1.0` fails fast with the existing compatibility UI.
- A gated call returning `AUTHZ-AUTHENTICATION-REQUIRED` triggers the session-expired channel.
- The mapper returns a defined outcome for every epic error code and a safe fallback for unknowns.
- No data/taxonomy read is issued before a session token is present.
- `retrieve` responses expose `ownerUuid` and `accessLevel` through the atom adapter/types.

## Non-goals

- Sign-up/sign-in/account UI (BI-260055/056/057/058+).
- Access-level **gating logic/affordances** (BI-260061) — only the `accessLevel`/`ownerUuid` field
  exposure is in scope here.
