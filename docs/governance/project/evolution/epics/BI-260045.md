# BI-260045: Add Flow-View Projection Mode Toggle for Non-Flow Atom Inclusion

**Status**: Ready
**Type**: Governance + Delivery
**Module**: workspace-graph, ui-shell
**Component**: Flow-view projection mode toggle and visibility semantics for non-flow atoms
**Blocks**: post-BI-260044 flow authoring ergonomics

## Dependencies

- ADR-260040: Flow View Projection Mode Toggle and Non-Flow Atom Visibility Refinement
- REQ-FR-260047: Flow View Projection Mode Toggle and Non-Flow Atom Inclusion
- REQ-FR-260045: Flow View Relayout Control and State Preservation
- REQ-FR-260046: Flow View Edge and Handle Visual Grammar

## Problem Statement

Strict Flow-view projection currently hides atoms without eligible flow bonds. This harms flow-authoring workflows when users need to create new dependency arrows from currently non-flow atoms.

## Acceptance Criteria

1. Flow view includes a binary projection mode control near existing Flow-view controls.
2. Default mode remains focused projection behavior (eligible bonds and participating atoms only).
3. Enabled mode includes non-flow atoms from the current search dataset while preserving Flow-view directional layout semantics.
4. In inclusion mode, non-flow atoms are visually distinguishable from flow-participating atoms.
5. Flow relayout remains left-to-right in both modes and preserves selected atom and viewport focus.
6. Directed flow edges keep visible arrowheads and continue suppressing rendered bond labels.
7. Tests validate toggle behavior, mode persistence rules, and no regression in existing Flow/Network boundaries.

## Verification

- Unit tests cover projection-mode filtering outcomes and node inclusion semantics.
- Interaction tests cover toggle state changes and relayout behavior in both modes.
- Rendering tests verify visual distinction of non-flow atoms and retained edge grammar.
- Regression tests verify Network view and existing flow controls are unaffected.

## Notes

- This item supersedes only the strict node-exclusion behavior from earlier Flow projection decisions.
- Orientation toggle (left-to-right vs top-to-bottom) remains deferred to future scope.
