# Epic: EPIC-260075 — Board View: kanban bulk classify by dimension

- Epic ID: EPIC-260075
- Title: Board View — kanban bulk classify by dimension
- Persona/Actor: Users classifying many atoms at once.
- Value statement: For users organizing many atoms, deliver a kanban board (columns = a dimension's values, cards = atoms, drag to recategorize) so bulk classification is fast.
- Context: §3 designed a board view for bulk classify; it was unscheduled until now. It registers a *view* (Center region), so it could not fold into the Classify inspector tab (EPIC-260067).
- Problem statement: There is no fast surface for reclassifying many atoms; doing it one atom at a time in the inspector is slow.
- Expected outcome: A **Board** center view — columns are the values of one chosen category dimension, atoms are cards, and dragging a card between columns recategorizes it. Registered as a center view; shares the selection model.
- Success metric(s): Users pick a dimension and bulk-reclassify atoms by dragging cards between value columns.
- Non-goals: Per-atom classify (EPIC-260067); category authoring (EPIC-260068).
- Constraints: Depends on category dimensions/values existing; consistent with the Q2 permission model when it lands.
- Dependencies: EPIC-260066 (view registry; category dimensions via shared widgets).
- Risks/assumptions: **Most deferrable leaf** — a convenience, not a pillar; safe to defer for a leaner v1.
- Target window: TBD (Ideation).
- Validation approach: Unit tests for column derivation + drag-to-recategorize; E2E dragging a card to a new value and confirming the category change.
