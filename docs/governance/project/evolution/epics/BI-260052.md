# Backlog Item: BI-260052 - Port Two-Column Auth + Demo Layout from Platform

- **Date**: 2026-06-10
- **Status**: Done

## Description

The crystord_platform marketing site has a well-designed sign-in page layout: a two-column panel
with a tabbed auth card (Sign In / Sign Up) on the left and a "Try Demo" feature card on the right.
The current `SignInPage` in the web app uses minimal inline styles with no visual hierarchy.

This backlog item ports the layout structure and interaction patterns from the platform into the
web app's `SignInPage`, using the web project's token system exclusively. Colors are not copied
from the platform; the web project's branding is the leading source.

## Goals

- Replace the flat SignInPage layout with a two-column auth + demo panel grid.
- Left column: tabbed interface (Sign In / Sign Up) with ARIA tablist, FormField styling, and
  "or sign in with" divider.
- Right column: "Try Demo" card with feature list (checkmarks in Control Green), primary CTA,
  and "No registration required" note.
- Responsive: stacks to a single column on narrow viewports (≤640px).
- All colors sourced from web project's CSS custom property tokens (no hardcoded hex).
- 100% unit test coverage for all new and updated components.
- New architecture tests document and enforce the layout constraints.
