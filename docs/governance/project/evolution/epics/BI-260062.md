# Backlog Item: BI-260062 - Align Label-Listing Read with the Schema 8.1.0 Operation Name

- **Date**: 2026-06-24
- **Status**: Done
- **Slice**: 8.1.0 baseline compatibility (surfaced during the BI-260054 review)

## Description

Schema 8.1.0 renamed the label-listing read from `list_labels(labels_prefix:)` to
`listLabels(labelsPrefix:)` (and the response field to `listLabels`). The client's `LIST_LABELS_QUERY`
still uses the old snake_case name, so under 8.1.0 the query fails GraphQL validation and breaks the
`workspace-search` recommended-labels read (pre-search discovery). This is a contract-alignment fix in
the same spirit as BI-260026 (bond selector type), not part of the auth/authz feature scope — but it is
required for the app to function against 8.1.0.

## Goals

- Update `LIST_LABELS_QUERY` to `listLabels(labelsPrefix: $prefix)` and `ListLabelsResponse` to expose
  `listLabels`.
- Update the consumer (`use-recommended-labels.ts`) to read `listLabels`.
- Keep the client-side variable name (`$prefix`) and call sites otherwise unchanged.

## Acceptance Criteria

- `LIST_LABELS_QUERY` requests `listLabels(labelsPrefix:)` matching `docs/crystord_server/schema.graphql`.
- Recommended labels populate from the `listLabels` response field.
- No remaining `list_labels` / `labels_prefix` reference in `src/`.

## Non-goals

- Taxonomy reads (`retrieveCategory*`) and other migration areas — separate epics.
- Any auth/authz behavior (BI-260054…BI-260061).

## Dependencies
