- Backlog Item ID: BI-260013
- Title: Formalize search, query, and discoverability model
- Persona/Actor: Data manager and domain operator
- Value statement: For graph-workspace users, define a predictable search and result-navigation model so discovery and focus workflows are reliable in MVP.
- Context: Search and discoverability decision phase for E1-E5.
- Problem statement: Without explicit E-section decisions, search UX may fragment between controls, query semantics, and result presentation, reducing operator efficiency.
- Expected outcome: Team can implement an MVP-safe search model with clear Post-MVP boundaries for advanced capabilities.
- Success metric(s):
  - E1-E5 are accepted and encoded in formal artifacts.
  - FR/CR requirements capture MVP behavior and deferral boundaries.
- Acceptance criteria (behavior):
  - ADR-260016 is created and accepted.
  - REQ-FR-260011, REQ-FR-260012, and REQ-CR-260005 are created and linked.
  - Product backlog and backlog status include BI-260013 with traceability.
- Non-goals: Full query builder in MVP, semantic autocomplete in MVP, and saved workspace-state persistence in MVP.
- Constraints: Preserve graph-first workflow and in-workspace navigation patterns.
- Dependencies: Accepted C and D decisions, backend GraphQL contract.
- Risks/assumptions:
  - Risk: under-scoped filters may reduce discoverability for medium-size datasets.
  - Risk: query summary may drift from actual backend request semantics if mapping is not enforced.
  - Assumption: label autocomplete and saved queries are sufficient first Post-MVP increments.
- Validation approach: End-to-end search workflow checks, UI consistency checks, and scope-gating checks.
- Priority score: 93
- Target window: MVP foundation
